## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| /admin route | Admin panel accessible to anyone who knows the URL |
| Database connection | PostgreSQL connection from app to AWS RDS |
| Auth session | Payload session cookies for admin users |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | Spoofing | /admin login | mitigate | Payload's built-in auth uses bcrypt password hashing and HTTP-only session cookies — no custom auth code needed |
| T-01-05 | Information Disclosure | /admin route | accept | Admin panel URL is publicly accessible but requires authentication. IP restriction deferred to Phase 4 (deployment/infrastructure concern). Low risk during development. |
| T-01-06 | Elevation of Privilege | User roles | mitigate | Users collection has role field (admin/editor). Payload's auth system enforces collection-level access control. Role-based field restrictions deferred to v2 (ADV-03). |
| T-01-07 | Tampering | Database connection | mitigate | DATABASE_URI uses sslmode=require for encrypted connection to AWS RDS as documented in .env.example |
</threat_model>
