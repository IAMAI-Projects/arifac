# Phase 02 — UI Design Contract

> Visual and interaction contract for Content Modeling & Frontend Integration. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase 2 objective:** Replace all hardcoded frontend content with CMS-fetched data. The visual design MUST remain identical -- no styling changes, no layout changes, no new UI chrome. This contract documents the existing visual system as a reference baseline and defines the NEW interaction states introduced by dynamic data fetching (empty, error, loading).

## Copywriting Contract

Phase 2 introduces NO new user-facing pages or CTAs. The CMS admin panel copywriting is handled by Payload's built-in admin UI. This contract covers the NEW frontend states that arise from dynamic data:

| Element | Copy |
|---------|------|
| Primary CTA | Not applicable -- existing CTAs (e.g. "Membership Benefits", "Explore Certifications") come from CMS data, not hardcoded in Phase 2 |
| Empty section (home page block missing) | Section is hidden -- not rendered (per D-12: strict empty state) |
| Empty collection (no regulatory updates) | Section is hidden -- not rendered |
| Empty page (no CMS document for slug) | Next.js `notFound()` -- renders default 404 page |
| Error state (CMS fetch failure) | Next.js `error.tsx` boundary: heading "Something went wrong", body "This page could not be loaded. Please try again later.", no retry button in v1 |
| News scroller empty | News scroller bar hidden entirely (per D-12) |

**Empty state strategy (D-12):** Missing or empty CMS content causes the section to not render. Missing pages (no document matching slug) return a 404. This is strict by design -- it forces editors to populate content.

**No destructive actions in this phase.** All destructive actions (delete content, restore versions) happen inside the Payload admin panel, which has its own UI.

## Interaction States

New states introduced by CMS data fetching:

### Page-Level Loading
- Server components fetch data before rendering -- no client-side loading spinners needed
- Pages either render with content or hit error/404 boundaries
- No skeleton screens in Phase 2 (server-rendered, not client-fetched)

### Certifications Filtering (Client Island)
- `CertificationsFilter` receives full certifications array from server parent via props
- Filter state is client-side `useState` (existing pattern preserved)
- Empty filter result: show "No certifications match your filters" centered in the grid area, text-neutral-500, text-[15px]
- Filter controls remain visible when no results match

### News Scroller
- If News Items collection returns 0 published items, the entire top bar in Header is hidden
- If 1+ items exist, marquee animation plays as current

### Home Page Blocks
- Each block type (hero, stats, capability matrix, regulatory dashboard, CTA) renders only if present in the page document's layout array
- Block order in admin panel determines render order on frontend
- Missing blocks = gap closes (no empty space left behind)
