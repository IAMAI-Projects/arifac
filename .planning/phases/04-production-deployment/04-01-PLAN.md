1. **Dockerfile** -- Create a multi-stage Dockerfile based on the official Payload CMS template. Use `node:22.17.0-alpine` as base. Three stages: deps, builder, runner.

```dockerfile
FROM node:22.17.0-alpine AS base

# --- Dependencies ---
FROM base AS deps
RUN apk add --no-cache libc6-compat
WORKDIR /app
COPY package.json pnpm-lock.yaml ./
RUN corepack enable pnpm && pnpm i --frozen-lockfile

# --- Builder ---
FROM base AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .

# Build args for Next.js build (NEXT_PUBLIC_ vars are inlined at build time)
ARG NEXT_PUBLIC_SERVER_URL
ENV NEXT_PUBLIC_SERVER_URL=${NEXT_PUBLIC_SERVER_URL}

# Use compile mode to avoid needing DATABASE_URI at build time
# All pages will be server-rendered at request time (appropriate for CMS)
ENV NEXT_TELEMETRY_DISABLED=1
RUN corepack enable pnpm && pnpm next build --experimental-build-mode compile

# --- Runner ---
FROM base AS runner
WORKDIR /app

ENV NODE_ENV=production
ENV NEXT_TELEMETRY_DISABLED=1

RUN addgroup --system --gid 1001 nodejs
RUN adduser --system --uid 1001 nextjs

COPY --from=builder /app/public ./public
RUN mkdir .next && chown nextjs:nodejs .next
COPY --from=builder --chown=nextjs:nodejs /app/.next/standalone ./
COPY --from=builder --chown=nextjs:nodejs /app/.next/static ./.next/static

USER nextjs

EXPOSE 3000
ENV PORT=3000
ENV HOSTNAME="0.0.0.0"

CMD ["node", "server.js"]
```

Key decisions:
- Uses `--experimental-build-mode compile` to skip SSG (no DATABASE_URI needed at build time). This is the Payload CMS recommended approach for Docker builds where DB is not accessible during build.
- NEXT_PUBLIC_SERVER_URL is passed as build arg because Next.js inlines NEXT_PUBLIC_ variables at build time.
- DATABASE_URI and PAYLOAD_SECRET are NOT build args -- they are runtime env vars passed via ECS task definition.
- Runs as non-root user `nextjs` (UID 1001) for container security.
- Uses `node:22.17.0-alpine` matching the official Payload template.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Build environment to Docker image | Build args (NEXT_PUBLIC_SERVER_URL) are inlined into the image; secrets must NOT be build args |
| Container to RDS | Database connection crosses network boundary; must use SSL |
| Internet to container (port 3000) | Health endpoint is publicly accessible via ALB |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | Dockerfile | mitigate | DATABASE_URI and PAYLOAD_SECRET are runtime env vars only, never build args; .dockerignore excludes .env files |
| T-04-02 | Information Disclosure | payload.config.ts | mitigate | SSL enabled on database connection in production via `ssl: { rejectUnauthorized: false }` in pool config |
| T-04-03 | Elevation of Privilege | Dockerfile | mitigate | Container runs as non-root user `nextjs` (UID 1001), not root |
| T-04-04 | Information Disclosure | health endpoint | accept | Health endpoint returns only `{ status: 'ok', timestamp }` -- no internal state, DB info, or version numbers exposed |
| T-04-05 | Denial of Service | health endpoint | accept | Health endpoint is lightweight (no DB calls); rate limiting handled by ALB/ECS |
</threat_model>
